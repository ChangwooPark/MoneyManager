# LINE通知 パートナー追加設定ガイド

取引保存時に本人とパートナーの両方にLINE通知を送るための設定方法です。

---

## 仕組みの概要

```
取引保存
   ↓
バックエンド (Cloud Run)
   ↓ LINE Multicast API
パートナー ←─── ボット ───→ 本人
```

パートナーのLINE User IDは **Webhook自動登録** 方式で取得します。
パートナーがボットにメッセージを送ると、バックエンドがUser IDをFirestoreに保存し、以降の取引通知が2人に同時送信されます。

---

## Step 1 — LINE Developersコンソール：Webhookを有効化

**https://developers.line.biz** → チャネル選択 → **Messaging API** タブ

| 項目 | 設定値 |
|------|--------|
| Webhook URL | `https://money-manager-1094294666571.asia-northeast3.run.app/notifications/line-webhook` |
| Webhookの利用 | **ON** |

URLを入力後、**検証**ボタンをクリック → `"成功"` を確認

---

## Step 2 — LINE公式アカウントマネージャー：自動応答をオフ

**https://manager.line.biz** → 該当チャネル → **応答設定**

| 項目 | 設定値 |
|------|--------|
| 応答モード | **Bot** |
| 自動応答メッセージ | **OFF** |

> **この設定は必須です。**
> 自動応答がオンのままだと、パートナーがメッセージを送ったときにLINEのデフォルトメッセージ
> （`このアカウントでは個別のお問い合わせは受け付けておりません`）が送られ、
> Webhookが動作せずパートナー登録ができません。

---

## Step 3 — パートナー登録（パートナー本人が実施）

パートナーは **LINE開発者アカウント不要** で、通常のLINEアプリだけで登録できます。

### ボットの共有方法
LINE Developersコンソール → Messaging APIタブの **QRコード** または **ボットID（@から始まる）** をパートナーに伝えます。

### パートナー登録手順
1. パートナーがLINEアプリでボットを **友だち追加**
2. ボットに **任意のメッセージを送信**（例：「登録して」、「こんにちは」）
3. ボットが自動返信：
   ```
   ✅ 通知受信者として登録されました！
   User ID: Uxxxxxxxxxxxxxxxxx
   ```

---

## Step 4 — 登録確認

アプリ → **その他** → **LINE通知** セクションを開き、受信者リストに **2名** 表示されていることを確認します。

以降、取引が保存されるたびに2人同時に通知が届きます。

---

## トラブルシューティング

| 症状 | 原因 | 解決方法 |
|------|------|----------|
| `このアカウントでは...` メッセージが届く | 自動応答がオン | Step 2 — 自動応答をOFF |
| 検証（Verify）失敗 | Webhook URLの誤りまたはCloud Run未デプロイ | URLを再確認、`git push`でデプロイ確認 |
| 登録後も通知が届かない | LINE通知トグルがOFF | その他 → LINE通知 → トグルON を確認 |
| 受信者リストに1名しか表示されない | パートナー登録が未完了 | Step 3を再試行 |

---

## 受信者の削除方法

アプリ → **その他** → **LINE通知** セクション → 受信者リスト → **削除** ボタン
